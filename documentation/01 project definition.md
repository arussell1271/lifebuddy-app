# 01 Project Definition.md

## The Holistic Cognitive Data Engine ("Life Buddy")



### Purpose

The "Why" and "What." The core business rules and requirements.



### What to Include

The expanded content from the initial prompt (Goals, Features, Audience).

##### 

##### 1\. Overall Goal (The Why)

To develop a web-based, and later mobile, application that implements the **Cultivate → Execute → Contribute** framework, integrating unconscious data with clinical health data to create a novel model for personalized, proactive patient care and **community impact research**.



##### 2\. Target User/Audience

**Primary Users:** Individuals committed to sustained behavioural change and health improvement.

**Secondary Users:** Linked Medical Professionals (e.g., "Dr. FRANK"), Mentors and Coaches, Research Entities (with explicit user consent).



##### 3\. Core Philosophical Foundation: Cultivate → Execute → Contribute

Sustained behavioral change is dependent on aligning one's internal self-image with desired actions and measurable, impactful results.

* **CULTIVATE:** Unconscious data (Dream Analysis, Spiritual Aspects) identifies the limiting subconscious identity/misalignment.
* **EXECUTE:** Introduction of **Holistic Actionable Items** (linked to habits) derived from the Cognitive synthesis.
* **CONTRIBUTE:** Improved health data (RHR, Sleep Score, Blood Results) and high adherence rates that **contribute to the user's overall wellbeing and the platform's research body.**



##### 4\. Core Features (Stage 1: The MVP - Core User Loop)

This stage establishes the essential data input, basic cognitive synthesis, and habit tracking.



###### | **Component**                  | **Description** |

| **Platform**                          | Desktop Web App (primary focus for initial build) and basic Mobile Web access. |

| **Unconscious Data Tracking**         | **CULTIVATE** (Identity Input)  | Dream Analysis \& Storing, Structured Spiritual Input. |

| **Actionable Items / Habit Tracking** | **EXECUTE** (Action Output)     | Daily Task List, Tracking of Holistic vs. Mandated Items. |

| **Health Analysis \& Feedback**        | **CONTRIBUTE** (Outcome Tracking)  | Storing and feedback on basic health data (RHR, Sleep Score, etc.). |

| **Basic Cognitive Synthesis**         | **CULTIVATE/EXECUTE Link**      | Cognitive agents interpret data to look for insights, linking unconscious data to conscious activity and suggesting/adjusting Actionable Items. |

| **Health Analysis**                   | Storing and feedback on basic health data (RHR, Sleep Score, etc.). Extracted From Apple Health Data |

| **Research Tracking**                 | Track anonymized data for research purposes and validation of the hypotheses (H1, H2, H3). |



##### 5\. Key Hypotheses (Guiding Principles for Cognitive Logic)

**\* H1 (EXECUTE Quality):** Adherence to Holistic (Identity-Focused) Items will result in a more sustainable improvement in health markers than Mandated (Action-Focused) Items alone.

**\* H2 (CULTIVATE Insight):** Frequency/theme of Unconscious/Spiritual Disalignment (Cultivate Data) will reliably predict Mandated Item non-adherence 3-5 days in advance.

**\* H3 (CONTRIBUTION Metric):** The Cognitive Engine's ability to provide a real-time, personalized **Cultivate → Execute → Contribution** path will result in significantly higher overall adherence rates (80%) compared to standard digital health tools.
